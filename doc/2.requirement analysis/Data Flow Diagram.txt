# Data Flow Diagram

## Introduction

A Data Flow Diagram (DFD) represents the movement of data through the system. It shows how housing data is collected, processed, analyzed, and displayed using Tableau dashboards.

## Data Flow

Step 1: Data Collection

The housing dataset is collected from a reliable source and stored in CSV or Excel format. The dataset contains information such as sale price, number of bedrooms, bathrooms, living area, lot area, location, and other property details.

Step 2: Data Import

The dataset is imported into Tableau Desktop for analysis and visualization.

Step 3: Data Processing

The imported data is cleaned and organized by removing duplicate records, handling missing values, and verifying data accuracy.

Step 4: Data Visualization

Tableau creates interactive charts, graphs, maps, and dashboards based on the processed data.

Step 5: User Interaction

Users interact with the dashboard using filters and charts to analyze housing market trends and compare different property features.

Step 6: Decision Making

Based on the dashboard insights, users can make better decisions regarding property buying, selling, or investment.

## Advantages

• Easy understanding of data movement.
• Improves system design.
• Provides a clear view of data processing.
• Supports efficient decision making.

## Conclusion

The Data Flow Diagram illustrates how housing data moves from collection to visualization, enabling users to analyze market trends through interactive Tableau dashboards.
